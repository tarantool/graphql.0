extra_args (db_schema, collection_name)
List of avro-schema fields to use as extra arguments of a collection / a connection field.

Mutation arguments (insert, update, delete) are generated here.

Parameters:

  • db_schema table e_schemas, schemas, collections, service_fields, indexes
  • collection_name string name of collection to create the fields

Returns:

  1. table list of avro-schema fields
  2. table

    map with flags to describe where generated arguments should be used; the format is the following:

    {
    <field name> = {
        add_to_mutations_only = <boolean>,
        add_to_top_fields_only = <boolean>,
    },
    ...
}

get_primary_key_type (db_schema, collection_name)
Get an avro-schema for a primary key by a collection name.

Parameters:

  • db_schema table e_schemas, schemas, collections, service_fields, indexes
  • collection_name string name of a collection

Returns:

  1. string offset_type is a just string in case of scalar primary key (and, then, offset) type
  2. table offset_type is a record in case of compound (multi-part) primary key (is not nil)
recursive_nullable (e_schema[, skip_cond])
Make schema types deep nullable down to scalar, union, array or map (matches xflatten input syntax) and remove default values (from fields of record and record* types).

Parameters:

  • e_schema (table or string) avro-schema with expanded references
  • skip_cond function (optional)

Returns:

    transformed avro-schema or nil (when the type or all its fields are matched by skip_cond)
recursive_replace_default_with_nullable (e_schema)
Remove default values from passed avro-schema (from fields of record and record* types) and make they nullable.

Returns:
